# HLTV Scraper

Scrapes CS match results from [HLTV.org](https://www.hltv.org) by year and saves map scores and player statistics to Parquet files. Includes an ETL pipeline to SQLite and an analysis notebook.

---

## How it works

The scraper runs a 3-level loop:

```
For each YEAR (e.g. 2024)
  └── For each results PAGE (offset 0, 100, 200 … until exhausted)
        └── For each MATCH on the page
              └── Opens the individual match URL
                    → parses match info, maps, and player stats
                    → saves to Parquet every N matches (checkpoint)
```

**Step 1 — results list**

Fetches `hltv.org/results?startDate=YYYY-01-01&endDate=YYYY-12-31&offset=N`.  
Each page returns up to 100 matches with: teams, final score, event name, and match URL.

**Step 2 — match detail (three independent parsers)**

Opens `hltv.org/matches/<id>/...` and runs three parsers on the same HTML:

| Parser | Output |
|---|---|
| `scraper_matches` | Match header: time, stage, format |
| `scraper_maps` | Per-map names and scores |
| `scraper_stats` | Per-player stats per map per side |

**Incremental checkpoints**

Every `SAVE_EVERY_N` matches the buffer is flushed to Parquet. If the scraper is interrupted, the next run reads already-saved `match_id`s and skips them automatically.

## Output structure

```
data/
  datasets/
    2012/
      matches.parquet       # one row per match
      maps.parquet          # one row per map played
      player_stats.parquet  # one row per player × map × side
    2013/
      ...
hltv.db                     # SQLite generated by etl.py
notebooks/
  analise_ano.ipynb
```

### matches.parquet

| Column | Type | Description |
|---|---|---|
| `match_id` | Int64 | Unique match ID |
| `date` | String | YYYY-MM-DD |
| `year` / `month` | Int32 | Derived from date |
| `match_time` | String | Start time (e.g. "17:00") |
| `team1` / `team2` | String | Team names |
| `score_team1` / `score_team2` | Int32 | Maps won |
| `event` | String | Event name |
| `stage` | String | Stage (e.g. "Quarter-Final") |
| `format` | String | Format (e.g. "bo3") |
| `match_url` | String | Full match URL |

### maps.parquet

| Column | Type | Description |
|---|---|---|
| `match_id` | Int64 | FK → matches |
| `map_order` | Int32 | 0 = "All Maps" aggregate, 1+ = individual maps |
| `map_name` | String | Map name or "all" |
| `score_team1` / `score_team2` | Int32 | Rounds won on this map |

### player_stats.parquet

| Column | Type | Description |
|---|---|---|
| `match_id` | Int64 | FK → matches |
| `map_order` | Int32 | Same as maps.parquet |
| `team` | Int32 | 1 or 2 |
| `side` | String | "both" / "ct" / "t" |
| `player_name` | String | Player nickname |
| `kills` / `deaths` | Int32 | |
| `adr` | Float64 | Average damage per round |
| `kast` | Float64 | KAST % (e.g. 72.7) |
| `rating` | Float64 | HLTV Rating 2.0 |

## Usage

```bash
# scrape a single year (resumes automatically if interrupted)
poetry run python main.py --year 2024

# scrape a range of years
poetry run python main.py --year-from 2012 --year-to 2026

# visible browser — needed to solve Cloudflare manually on first run
poetry run python main.py --year 2024 --no-headless

# verbose debug logs
poetry run python main.py --year 2024 --debug

# wipe existing data and re-scrape from scratch
poetry run python main.py --year 2024 --force-download

# reprocess only one part without re-scraping everything
poetry run python main.py --year 2024 --parts maps
poetry run python main.py --year 2024 --parts stats
poetry run python main.py --year 2024 --parts maps stats
```

> **Cloudflare:** on the first run HLTV may show a challenge page. With `--no-headless` the browser opens visibly — solve the captcha manually. Cookies are saved to `hltv_cookies.json` and reused on subsequent runs.

---

## Partial reprocessing (`--parts`)

The three output files are generated by independent parsers and can be reprocessed individually:

```
matches.parquet  ← requires iterating results pages (full scrape)
maps.parquet     ← requires only match URLs from matches.parquet
player_stats.parquet ← requires only match URLs from matches.parquet
```

Typical workflow when adding a new field to maps:

```bash
# 1. Update scraper_maps.py  →  add the new field
# 2. Reprocess only maps for the affected year (matches.parquet is the URL source)
poetry run python main.py --year 2024 --parts maps --force-download
```

## ETL — Parquet → SQLite

```bash
poetry run python etl.py              # load all years into hltv.db
poetry run python etl.py --year 2024  # only a specific year
poetry run python etl.py --reset      # rebuild from scratch
```

SQLite tables: `matches`, `maps`, `player_stats` (mirror of the Parquet schemas above).

## Debug tools

```bash
# download the HTML of one specific match for selector inspection
poetry run python scripts/debug_fetch_match.py 2209322
poetry run python scripts/debug_fetch_match.py https://www.hltv.org/matches/2209322/...

# download a sample of match pages per year (recent + oldest + random)
poetry run python scripts/debug_sample_matches.py             # all years
poetry run python scripts/debug_sample_matches.py --year 2024 # single year
```

HTML files are saved to `data/debug_match/<match_id>.html`.

## Analysis notebook

Open `notebooks/analise_ano.ipynb`. Change the `YEAR` variable in the first cell to analyse any collected year. Requires the kernel:

```bash
poetry run python -m ipykernel install --user --name hltv-scraper
```
