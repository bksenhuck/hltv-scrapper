# HLTV Scraper

Scrapes CS match results from [HLTV.org](https://www.hltv.org) by year, saving map scores and player statistics to Parquet files. Includes an ETL pipeline to SQLite and an analysis notebook.

---

## How it works

The scraper runs a 3-level loop:

```
For each YEAR (e.g. 2012)
  └── For each results PAGE (offset 0, 100, 200 ... until exhausted)
        └── For each MATCH on the page
              └── Opens the individual match URL and extracts full details
                  → saves to Parquet every 50 matches (checkpoint)
```

**Step 1 — results list**

Fetches `hltv.org/results?startDate=YYYY-01-01&endDate=YYYY-12-31&offset=N`. Each page returns up to 100 matches with: teams, final score, event name, and match URL.

**Step 2 — match detail**

Opens `hltv.org/matches/<id>/...` and extracts:
- Match time, stage (Quarter-Final, etc.), format (Best of X)
- For each map: name and score per team
- For each player on each map: kills, deaths, ADR, KAST%, HLTV rating

**Incremental checkpoints**

Every 50 matches the buffer is flushed to Parquet. If the scraper crashes, the next run reads already-saved `match_id`s and skips them automatically.

## File structure

```
datasets/
  2012/
    01.parquet   # matches from January 2012
    02.parquet
    ...
  2024/
    ...
hltv.db          # SQLite database generated by the ETL (tables: matches, maps, player_stats)
notebooks/
  analise_ano.ipynb   # exploratory analysis by year
```

**Parquet schema:**

| Field | Type | Description |
|---|---|---|
| `match_id` | Int64 | Unique match ID |
| `date` | String | Date (YYYY-MM-DD) |
| `match_time` | String | Match start time |
| `team1` / `team2` | String | Team names |
| `score_team1` / `score_team2` | Int32 | Final score (maps won) |
| `event` | String | Event name |
| `stage` | String | Stage (e.g. Quarter-Final) |
| `format` | String | Format (e.g. Best of 3) |
| `maps_json` | String | JSON with per-map scores and player stats |
| `match_url` | String | Full match URL |

## Installation

```bash
# install dependencies
poetry install

# download the Firefox binary used by camoufox
poetry run camoufox fetch
```

---

## Usage

```bash
# scrape a single year
poetry run python main.py --year 2024

# scrape a range of years
poetry run python main.py --year-from 2012 --year-to 2024

# visible browser — required to solve Cloudflare manually
poetry run python main.py --year 2024 --no-headless

# verbose debug logs
poetry run python main.py --year 2024 --debug
```

> **Cloudflare:** on the first run HLTV may show a challenge page. With `--no-headless` the browser opens visibly — solve the captcha manually and press Enter in the terminal to continue. Cookies are saved to `hltv_cookies.json` and reused on subsequent runs.

---

## ETL — Parquet → SQLite

```bash
# generate / update hltv.db from all collected data
poetry run python etl.py

# only a specific year
poetry run python etl.py --year 2024

# rebuild the database from scratch
poetry run python etl.py --reset
```

Generated tables:
- `matches` — one row per match
- `maps` — one row per map played
- `player_stats` — one row per player per map

## Analysis notebook

Open `notebooks/analise_ano.ipynb` in VS Code or Jupyter. Change the `YEAR` variable in the first parameter cell to analyse any collected year.

To register the project kernel in VS Code:

```bash
poetry run python -m ipykernel install --user --name hltv-scraper
```
